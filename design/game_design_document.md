# RRaM - геймдизайн-документ

## 1. Концепция

RRaM - онлайн RTS/RPG игра, построенная вокруг управления группой персонажей на островной карте. Игроки начинают на противоположных островах, перемещаются по ромбообразной сетке бордов, используют карты, передают ресурсы между персонажами и применяют специальные эффекты вроде телепортации.

## 2. Игровая карта

Карта состоит из островов и черной ромбообразной сетки. Ромбообразные ячейки называются бордами. Персонажи перемещаются по бордам во время хода.

### 2.1 Стартовые острова

Основные персонажи игроков начинают игру на противоположных островах. На карте должны быть подписанные стартовые ячейки для каждого типа персонажа.

### 2.2 Подписанные ячейки

Обозначения стартовых ячеек:

- `К` - кузнец.
- `П` - помощник кузнеца.
- `В` - воин.
- `О` - охотник.
- `Ш` - шаман.

При расстановке игры персонажи, которые передвигаются по карте, начинают свои ходы с соответствующих подписанных мест.

### 2.3 Точки телепортации

Подписанные стартовые ячейки также используются как точки телепортации. Например, при использовании карты `Бусы телепортации` персонаж может переместиться на одну из этих подписанных точек, если это разрешено описанием карты.

## 3. Персонажи

Базовый набор персонажей:

- Кузнец.
- Помощник кузнеца.
- Воин.
- Охотник.
- Шаман.

Каждый персонаж является отдельной игровой единицей, может находиться на карте и взаимодействовать с картами, бордами и другими персонажами.

## 4. Ходы и кубики

Каждому игроку в начале игры дается 10 бросков.

В рамках броска используются два кубика. Значения кубиков можно применять несколькими способами:

- Сложить значения двух кубиков и передвинуть персонажа на суммарное количество бордов.
- Использовать один кубик для передачи карт от одного персонажа другому.
- Использовать второй кубик для добора карты из колоды.
- Использовать второй кубик для перемещения персонажа.

## 5. Передача карт

Если игрок использует кубик для передачи карт, значение на кубике определяет максимальное количество карт, которые можно передать.

Пример: если на кубике выпало `2`, игрок может передать от одного персонажа другому до двух карт.

## 6. Добор карт

Если игрок использует кубик для добора карты, значение на кубике не определяет количество добираемых карт.

Пример: если на кубике выпало `5`, игрок все равно берет только одну карту из выбранной разрешенной колоды.

## 7. Открытые вопросы

- Сколько игроков участвует в одной партии?
- Сколько персонажей у каждого игрока на старте?
- Все ли персонажи могут носить карты?
- Есть ли ограничение руки или инвентаря?
- Что считается одним ходом в онлайн-версии: бросок, серия из 10 бросков или раунд?
- Какие существуют колоды?
- Можно ли телепортироваться на стартовые точки противника?
- Можно ли занимать одну борду несколькими персонажами?
- Есть ли бой при встрече персонажей на соседних или одинаковых бордах?
