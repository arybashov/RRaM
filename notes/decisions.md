# RRaM - утвержденные решения

Этот файл нужен, чтобы фиксировать решения после обсуждения и не возвращаться к одним и тем же вопросам.

## Уже принято

- Проект называется `RRaM`.
- Формат: онлайн RTS/RPG.
- Игроки начинают на противоположных островах.
- На карте есть ромбообразная сетка бордов.
- Основные стартовые персонажи:
  - `К` - кузнец.
  - `П` - помощник кузнеца.
  - `В` - воин.
  - `О` - охотник.
  - `Ш` - шаман.
- Подписанные стартовые точки используются для начальной расстановки.
- Эти же точки могут быть целями телепортации через карту `Бусы телепортации`.
- Каждому игроку изначально дается 10 бросков.
- Один бросок включает два кубика.
- Два кубика можно сложить для движения.
- Один кубик можно потратить на передачу карт.
- Второй кубик можно потратить на добор карты или движение.
- При передаче карт значение кубика задает лимит карт.
- При доборе карты значение кубика не увеличивает количество карт: берется одна карта.

## Решения для прототипа (5 июня 2026)

Зафиксировано в обсуждении. Часть пунктов завязана на реальную карту — карту пришлет заказчик, до этого правила движения и победы не вшиваем жестко.

- Матч `1 на 1`, строго пошаговый. HP каждого персонажа: 100.
- Дварфы (3-й активный игрок по правилам) — в прототипе не реализованы, запланированы как ИИ-слот в будущем. `PLAYER_LIMIT = 2`.
- Ход = один бросок двух `d6`; 10 бросков на партию на игрока, не пополняются.
- Хранение карт: инвентарь у персонажа, общей руки нет. Лимит инвентаря: 10 карт (по PDF-правилам).
- Передача карт: только между персонажами своего игрока на одной борде.
- Колода: конечный перемешанный стек; добор берет 1 карту с верха.
- Занятость борда: один персонаж; занятые борды блокируют путь и телепорт.
- Движение: «до N» (от 1 до значения кубика/суммы), путь по соседним гексам без прохода сквозь занятые. *(Ждет реальную карту с проходимостью.)*
- Телепорт `Бусами`: только на свои стартовые точки, цель не занята.
- Условие победы прототипа: гонка — привести любого персонажа на стартовую точку противника. *(Ждет реальную карту со стартовыми точками обоих игроков.)*

## Архитектура (5 июня 2026)

- **Сервер — единственный источник истины.** Вся игровая логика (движение, карты, телепорт, ходы, победа) живет на сервере.
- Клиенты тонкие: шлют *намерения* (intents), сервер валидирует и рассылает новое состояние. Текущий веб-прототип и будущий 3D-клиент — только отрисовка и ввод.
- Кубики бросает сервер.
- Скрытая информация (рука/инвентарь): сервер шлет персональные снимки, противник не видит чужие карты.
- Состояние в памяти процесса для прототипа; БД позже.
- Движок правил — отдельный модуль без сетевого кода, чтобы переиспользовать.
